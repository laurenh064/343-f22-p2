Our app lets the user search images from the Met Museum and generate color palates for the selected image. The users can filter results based on the department of the art.

  • deploy site - your site (or "app") for this project is deployed publicly (e.g. on github.io or w3.cs.jmu.edu)

Our side is published to github for viewing.

  • is responsive - your site has varying presentation on mobile and on larger viewports (e.g "desktop") accomplished through use of media queries

We used bootstrap to be able to have an adaptive navbar which compresses with a smaller screen. The search results shrink to fit the screen and if you click on a picture it will properly fill the space.

  • uses 2 different apis - use at least 2 existing APIs to create some kind of functionality that you like

We used the Metropolitan Museum of Art Met Collection API to find art pieces by searching for titles, artists, or composition. Once we had the piece of art, we used Imagga's Image Recognition API to extract a color palate from the image of the chosen piece of art to display.

  • has search - your app should have some kind of search

We built a search bar to use for finding your piece of art. You can search for a title, artist, keyword, country, time period, or anything specific you are trying to find.

  • displays results - your app should display the results to the search

Our search results are displayed in a masonry layout to the screen, and there is a spin box to specify how many results are on the page at a time.

  • results are filterable - the visitor should be able to interact with the results at least being able to filter them, possibly sorting or other manipulations

We have a dropdown menu where you can pick the department to display and it will only show the art pieces that are in that department. You can also select all to have all the results displayed.
